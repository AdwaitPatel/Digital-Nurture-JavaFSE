# Exercise 2

## Big O Notation
- Big O measures how an algorithm performs as input size increases.
- Used to compare the efficiency of algorithms.

## Search Operations

### Linear Search
- Best Case: O(1)
- Average Case: O(n)
- Worst Case: O(n)

### Binary Search
- Array must be sorted.
- Best Case: O(1)
- Average Case: O(log n)
- Worst Case: O(log n)

## Time Complexity Comparison

| Algorithm | Best | Average | Worst |
|-----------|------|---------|-------|
| Linear Search | O(1) | O(n) | O(n) |
| Binary Search | O(1) | O(log n) | O(log n) |

## Conclusion
- Linear Search is simple and works on unsorted arrays.
- Binary Search is faster but requires a sorted array.
- Binary Search is more suitable for an e-commerce platform because it provides faster search performance for large datasets.
